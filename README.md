# ETF Rebalance

一个完全在浏览器本地运行的中文 ETF 定投工具。输入持仓、价格、目标比例和本月资金后，它会同时给出理论分配与可实际下单的整数手购买建议。

## 打开方式

无需安装依赖或构建：

1. 直接双击 `index.html`，适合日常离线使用。
2. 或在本目录运行静态服务器：

   ```bash
   python3 -m http.server 8765
   ```

   然后打开 <http://localhost:8765>。

## 安装到 iPhone

PWA 需要通过 HTTPS 地址打开，`file://` 双击模式仍可使用计算器，但不能安装或启用离线缓存。

1. 把整个目录发布到任意静态 HTTPS 地址（GitHub Pages、Cloudflare Pages、Netlify 等均可）。
2. 在 iPhone 的 Safari 中打开该地址。
3. 点击 Safari 的“分享”按钮，选择“添加到主屏幕”，名称会显示为 **ETF Rebalance**。
4. 从主屏幕首次打开后，页面外壳会保存在手机中；以后没有网络也能启动、计算和读取本机保存的数据。

行情接口不会写入离线缓存：离线时仍可使用已有价格或手工输入价格，但“刷新全部行情”需联网。页面发现新版本时会显示“立即更新”，由用户确认后刷新，既有 `localStorage` 数据不会被清空。建议定期使用“导出配置”另存 JSON 备份。

## 使用方法

1. 填写当前现金、本月新增资金、预留现金、佣金费率和每笔最低佣金。当前现金是账户已有或上月结转资金；本月新增是本月准备额外投入的资金。
2. 为每项资产填写持有份数与证券代码，点击“刷新全部行情”获取沪深 ETF/A 股最新价格，也可手工修改价格。目标比例可小于或等于 100%，未分配比例会作为现金目标；不能超过 100%。
3. 点击“生成本月方案”，查看理论模式、整手模式、剩余现金和偏差变化。
4. 整手模式会同时显示系统建议和“实际手数”。可按真实下单计划修改实际手数，金额、佣金、剩余现金和买后比例会重新计算。
5. 实际下单完成后点击“确认已执行”，持仓与现金会按实际手数更新，并写入执行历史。

手机端的资产默认以摘要卡片折叠显示，点击卡片可展开编辑；资产列表底部保留“添加资产”按钮，资产较多时无需返回列表顶部。

初次打开时只提供一个目标比例为 100% 的空白资产，不预置任何证券名称或代码。名称、代码、份数、价格、目标、每手份数均可修改，也可暂停、添加或删除资产。

配置与历史记录会自动保存在当前浏览器的 `localStorage`。可用“导出配置”备份为 JSON，再通过“导入配置”恢复；执行历史可导出为带 UTF-8 BOM 的 CSV，方便用 Excel 打开。重置会恢复默认资产并清空本地历史，执行前会再次确认。

## 算法说明

- 理论模式根据投入后的目标市值缺口分配资金，允许零碎份额，不计整手和佣金。
- 整手模式只考虑未暂停且价格有效的资产。算法每轮模拟购买一手，在预算可承受的候选中选择使目标比例平方误差下降最多的一项。
- 同一资产的建议视作一笔交易，佣金为 `max(成交金额 × 佣金费率 / 10000, 单笔最低佣金)`，多手只计算一笔佣金。购买数量始终是“每手份数”的整数倍；不足一手、无法继续改善偏差或预留的资金都会保留为现金。
- 工具不卖出超配资产，而是用新增资金逐步稀释其比例。

## 测试

需要 Node.js 18 或更高版本，测试只使用 Node 内置测试运行器：

```bash
node --test tests/*.test.js
```

核心逻辑位于 `calculator.js`，同时支持浏览器全局变量 `ETFCalculator` 与 Node.js `require()`，不依赖 DOM 或外部包。

## 数据与风险

本项目没有后端或第三方依赖。行情通过腾讯公开沪深行情接口获取；公开接口未来可能变化，失败时页面会保留原价格，用户仍可手工输入。所有配置和记录仅保存在浏览器本地（除非主动导出文件）。

本工具只做数学计算，不提供行情或投资建议。市场有风险，投资需谨慎。
