# ETF 再平衡计算器：资产总览、行情与佣金设计

日期：2026-07-12

## 目标

在保持纯 HTML、CSS、JavaScript、无后端、可直接双击打开的前提下，增强现有 ETF 定投再平衡计算器：在页面顶部展示当前资产全貌，澄清现金字段语义，加入按比例计算的佣金，并为沪深 ETF 与 A 股提供按证券代码获取最新行情的能力。

## 范围

- 支持沪深交易所 ETF 和 A 股代码，不支持港股、美股或场外基金。
- 保留现有理论模式、整手模式、localStorage、导入导出、执行历史与确认执行流程。
- 不新增后端、构建工具或第三方运行时依赖。
- 不计算印花税、过户费及卖出费用；当前产品只生成买入建议。

## 页面结构

### 顶部资产总览

在现有标题区下方、资金设置上方新增“目前资产情况”。总览包含四张摘要卡：

1. 持仓证券总市值；
2. 当前现金；
3. 本月新增资金；
4. 总资产。

总资产定义为：

```text
总资产 = 全部持仓市值 + 当前现金 + 本月新增资金
```

摘要卡下展示已持仓证券的紧凑列表，仅列出持有份数大于零的资产。字段为代码、名称、持有份数、最新价格、市值和当前占比。没有持仓时显示明确的空状态。资产输入、价格刷新或资金设置变化后，总览立即更新。

### 资金设置

字段保留“当前现金”和“本月新增”，并增加常驻说明：

- 当前现金：账户中已经存在、通常来自以前月份结转的可用现金；
- 本月新增：本月计划额外投入、确认执行后清零的资金。

开启“使用当前现金参与本次再平衡”时：

```text
可用预算 = 当前现金 + 本月新增 - 预留现金
```

关闭时：

```text
可用预算 = 本月新增 - 预留现金
```

若预留现金超过本次参与计算的资金，显示中文校验错误，不把负数静默截断为零。

### 佣金设置

增加“佣金费率（万分之）”字段，保留“单笔最低佣金”。每项资产的最终建议视为一笔交易：

```text
单笔佣金 = max(该资产总成交金额 × 佣金费率 / 10000, 单笔最低佣金)
```

佣金只对实际产生的买入计算，并保留两位小数。同一资产买入多手只收一笔最低佣金。算法必须确保成交金额与佣金合计不超过可用预算。

## 行情获取

### 数据源与调用方式

使用腾讯沪深行情接口作为默认数据源。该接口已验证可以通过 HTTPS 返回沪深行情并允许跨域访问，适合从本地 `file://` 页面调用。证券代码映射规则：

- `5`、`6`、`9` 开头映射为上海市场前缀 `sh`；
- `0`、`1`、`2`、`3` 开头映射为深圳市场前缀 `sz`；
- 只接受六位数字代码。

行情访问封装为独立模块接口，使未来更换数据源时不影响页面与计算逻辑。

### 交互

- 每行资产提供“获取行情”按钮；
- “资产与目标”标题区域提供“刷新全部行情”按钮；
- 输入或修改为有效代码后允许主动获取，不在每次键入时自动请求；
- 成功后更新资产名称、最新价格和行情时间，并立即保存到 localStorage；
- 价格输入框继续允许手工覆盖；
- 显示最近行情时间及成功、失败或刷新中的状态。

### 降级与错误处理

公开行情接口不是正式承诺稳定性的开放 API，因此页面必须可降级：

- 网络或接口失败时保留原价格，不清空数据；
- 单项失败不阻止其他资产更新；
- 全部刷新后汇总显示成功与失败数量；
- 无效代码、未上市代码、接口无数据、网络失败分别显示可理解的中文提示；
- 行情无法获取时，用户仍可手工输入价格并完成计算。

## 数据模型与迁移

全局设置新增：

```js
commissionRate: 0
```

资产新增：

```js
quoteUpdatedAt: ''
```

现有 localStorage 使用相同主键并在加载时补齐默认字段，以无损迁移旧数据。导出 JSON 包含新增字段，导入旧版 JSON 时自动补默认值。

## 确认执行后的现金

确认执行后：

1. 建议买入份数加入对应持仓；
2. 本月新增资金清零；
3. 实际支出包含成交金额与佣金；
4. 未使用资金结转到当前现金；
5. 若关闭“使用当前现金”，原当前现金保持不变，只有本月新增的未使用部分结转；
6. 保存更新后的持仓、现金和历史记录。

统一计算式为：

```text
执行后当前现金 = 当前现金 + 本月新增 - 实际支出
```

“是否使用当前现金”只限制算法本次可动用的预算，不改变账户总现金的会计关系。

## 模块边界

- `calculator.js`：纯计算、校验、佣金函数和再平衡算法；不访问 DOM 或网络。
- `quotes.js`：代码校验、市场前缀映射、行情响应解析和网络请求。
- `app.js`：状态迁移、事件绑定、行情刷新编排、localStorage 与页面渲染。
- `index.html` / `styles.css`：资产总览、字段说明、行情状态及响应式布局。

## 测试与验收

自动测试覆盖：

- 总资产与持仓列表所需的汇总数据；
- 开启和关闭当前现金参与时的预算差异；
- 预留现金超过参与资金时返回明确错误；
- `max(比例佣金, 最低佣金)` 规则；
- 多手同一资产只计算一次最低佣金；
- 建议总成本不超过预算；
- 上海、深圳代码映射和非法代码拒绝；
- 行情响应解析；
- 行情失败时不覆盖旧价格；
- 新字段的 localStorage 迁移与持久化；
- 现有再平衡测试全部回归通过。

浏览器验收覆盖：

- 通过本地静态服务器打开页面并填入有效持仓；
- 获取单只行情及刷新全部行情；
- 验证顶部数值随持仓、价格和现金实时变化；
- 验证计算结果及确认执行后的现金结转；
- 检查桌面宽度与手机宽度布局；
- 检查浏览器控制台无 JavaScript 错误。

## 风险与明确取舍

- 腾讯行情接口可能调整或停止服务；手工价格和独立行情模块是必要兜底。
- 双击 `file://` 页面可使用行情接口，但不同浏览器的网络安全策略可能变化；README 同时提供本地静态服务器打开方式。
- 行情只用于填充用户输入，不构成行情服务承诺，也不改变页面底部的投资风险提示。
